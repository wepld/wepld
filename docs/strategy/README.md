## Product definition

WePLD is the governed engineering operating system that turns compatible human
and AI engineering capacity into a complete, evidence-driven, learning
engineering organization. It is not an AI editor, a multi-agent chat room, a
model router, a terminal wrapper, a prompt library, a Cursor clone, a
collection of agent personas, or a replacement name for Hermes.

The intended user journey: connect a repository and compatible runtime →
describe an outcome → resolve material ambiguity → approve a specification →
approve a qualified plan → allow bounded execution → inspect only material
decisions → receive an evidence-backed completion proposal → retain verified
reusable engineering intelligence.

## Permanent authority principles

WePLD Core owns durable engineering truth. Authenticated humans remain final
authority. AGILLE owns the governed delivery method. Hermes provides
engineering intelligence and execution capacity. Agent Hive organizes bounded
specialist roles. The Committee produces advisory deliberation only. SkillHouse
stores evaluated reusable capabilities. Engineering Memory is advisory and
governed. Effects pass through the Effect Firewall. Completion requires
evidence and an authorized decision. A model, agent, Committee, score, vote, or
reputation never becomes authority. Honest `Blocked`, `Deferred`, `Returned`,
`Unsafe`, `Rejected`, and `NonConvergent` outcomes are valid. Consumer chat
subscriptions are not treated as APIs; WePLD must not capture browser cookies,
automate consumer chat sessions, or circumvent provider usage restrictions.
Every planned feature remains unauthorized until its stage gate is approved.
