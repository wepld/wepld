From described outcome to verified acceptance

Core owns durable specification, policy, approvals, plans, WIP, effects, evidence requirements, completion, and recovery. Brains, Hermes, builders, and subagents cannot approve themselves or mutate mission truth.

Authority invariant: governance policy → approved specification → Outcome Contract → delivery plan → phase plan → task packet → tool action. A Brain proposes; Core authorizes; the effect boundary executes and probes; evidence verifies; the user accepts where required.

Governed delivery, not one-pass coding

The native method separates WHAT from HOW, then runs bounded evidence-producing loops. Changes to WHAT create a new specification version; changes to HOW use controlled replanning.

Trust-first roadmap

Historical M0–M7 remains lineage. Current delivery uses a separately named baseline gate and H1–H9 so a polished shell cannot outrun runtime truth.

Build Feature Baseline Gate

Record Draft PR #1 as accepted, returned, deferred, or rejected, then resolve the gate through accepted prerequisite contracts or an approved replacement-foundation plan. No disposition ratifies or authorizes its merge, and non-acceptance cannot deadlock H1.

Early spine + H1–H2

Before H1/H2, establish exact-provenance runs and milestone baselines. Qualify PlanProposal → candidate → PlanAssessment → risk-tier review → PlanDecision; design typed SOP and authorized role-subscription graphs.

H3.1/H3.2 + H4.1/H4.2/H4.3

Prove built-in skills/hooks, capability-projected tools, bounded results and SOP execution before optional packaging. Prove exploration/compaction plus reproducible context before structural/semantic extensions.

H5–H6 — Controlled loops and roles

Use typed sandbox denials, test an advisory-only risk advisor, replan/escalate/stop under budgets, and supervise authorized role subscriptions, structured handoffs, isolated writes, and review.

H7–H8 — Typed memory and outcome convergence

Judge branch/compaction-derived learning. H8 consumes accumulated runs, then tests controlled cross-model, repeated/randomized, ablation, multi-route-race, independent-scoring, and certification evidence.

H9 — Product surfaces after runtime truth

Mission Control and Engineering-Team UX visualize execution, routes, artifacts and Evidence Bundles over one Core workflow; views never become authority or acceptance evidence themselves.
