Business context

WePLD is not an AI IDE. The editor is one workspace in a studio that coordinates planning, building, review, testing, security, knowledge, executive decisions, and releases.

From goal to verified outcome

The Core Daemon makes the organization’s work durable and causal. Workers never directly message the human or mutate mission state.

Key invariant: a Brain can propose an action, but only policy can permit it and only the Tool Executor can perform it. A worker reports artifacts; the Orchestrator alone advances task and mission state.

Iterative engineering, not one-pass coding

Every mission travels through an evidence-producing loop. Stages are governed by task type and risk; completion is a policy-evaluated fact, not a worker declaration.

System architecture

The V1 deployment is intentionally modest: a trusted local control plane, isolated execution, and explicit adapters around external systems. Clean ports make future cloud/enterprise evolution possible without bringing distributed-systems complexity into the first mission.

Local-first deployment and trust boundaries

Trust-first roadmap

The delivery order prevents an attractive shell from outrunning the safety and recovery model.

M0 — Architecture gate

Approve scope, contracts, threat model, evaluation fixtures, and technical spikes. Production implementation remains blocked until this gate clears.

M1/M2 — Durable local core + safe task effect

Prove authoritative recovery, authenticated local RPC, policy enforcement, event projections, isolated worktrees, and one capability-controlled effect.

M3 — First full mission loop

Planner → Builder → QA/Reviewer completes a bounded local mission with a reviewable change, decision queue, and evidence timeline.

M4/M5 — Governed autonomy + memory

Add bounded parallelism, retries, budgets, gates, Mission Control health, skills, knowledge, and full-autonomy envelopes.

M6/M7 — Studio beta, then ecosystem

Expand workspaces, package lifecycle, selected integrations, release operations, and only then enterprise/remote collaboration.
